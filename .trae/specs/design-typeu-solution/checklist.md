# Checklist

## 项目结构与技术栈

- [ ] 解决方案包含 8 个项目（Core/Models/Network/Data/Services/Teacher.GUI/Student.GUI/Tests），命名遵循 `TypeU.{Layer}` 约定
- [ ] `Directory.Build.props` 统一配置 .NET 10、Nullable enable、TreatWarningsAsErrors、LangVersion latest
- [ ] 禁用顶级语句，所有入口使用标准 Main 方法
- [ ] 所有代码命名空间以 `Larpx.PersonalTools.TypeU.` 开头，命名空间包裹所有代码
- [ ] NuGet 包均为最新稳定版（Avalonia 11、protobuf-net、Dapper、Microsoft.Data.Sqlite、MiniExcel、Serilog、CommunityToolkit.Mvvm）
- [ ] AOT 发布属性配置正确（PublishAot=true、StripSymbols=true、DebugType=none 不生成 PDB）
- [ ] `dotnet build` 通过，0 错误 0 警告

## 通信协议与网络层

- [ ] 报文结构完整实现：Magic(2B,0xAA55) | Version(1B) | MsgType(2B) | Timestamp(8B) | Nonce(16B) | DataLen(4B) | Data(NB) | Signature(32B) | Tail(2B,0x55AA)
- [ ] Data 字段使用 AES-256-CBC 加密
- [ ] HMAC-SHA256 签名覆盖 Header+Data，校验失败的报文被丢弃并记录告警
- [ ] 时间戳偏差 > 60 秒的报文被拒绝
- [ ] Nonce 缓存窗口 120 秒，重复 Nonce 被拒绝
- [ ] 教师端定时（10 秒）广播 `TimeSyncMessage` 给所有在线学生端，作为考试倒计时唯一时间基准
- [ ] 学生端接收教师端时间后忽略本地系统时间修改，按教师端时间计算考试起止
- [ ] 教师端定时（5 秒）通过 UDP 广播自身 IP+TCP 端口，启动监听即开始广播
- [ ] 学生端启动时监听 UDP 广播自动发现教师端，5 秒内完成连接
- [ ] 自动发现失败时学生端支持手动输入教师端 IP 兜底
- [ ] TCP 服务端支持 100 并发连接稳定
- [ ] TCP 客户端实现断线自动重连，重连后补传未上报数据
- [ ] 协议编解码、签名校验、防重放拦截、时间同步、自动发现均有单元测试覆盖

## 教师端服务端能力

- [ ] 一键启动局域网监听并自动发现在线学生
- [ ] 局域网一键扫描发现所有在线电脑（IP/MAC/计算机名/是否已安装学生端），可手动添加未识别设备 IP
- [ ] 学生列表显示 IP、计算机名、学号、姓名、状态
- [ ] 题库支持增删改查与 TXT 文件导入
- [ ] 三种考试模式实现：定篇测速、限时冲刺(1-20 分钟)、纠错模式
- [ ] 开始考试时加密下发试题，学生端进入锁定沉浸模式
- [ ] 暂停/停止/重新考试流程控制正确（重新考试需重置学生端状态与本地草稿）
- [ ] 监控仪表盘状态色块正确（灰-离线/绿-考试中/蓝-已完成/红-异常）
- [ ] 异常浮窗滚动显示作弊嫌疑警报
