**文档版本**：V1.0  
**项目名称**：TypeU
**目标平台**：Windows 7/10/11, Linux (x64)  
**编写日期**：2026-07-21
---

## 1. 项目概况
### 1.1 项目背景
本系统旨在解决大学机房环境下打字教学与考核的电子化需求。传统打字测试软件多为单机版，缺乏统一管理、实时监控及有效防作弊手段。本系统通过局域网互联，实现教师端对考试全流程的数字化控制。
### 1.2 项目目标
*   **跨平台运行**：基于 .NET 框架开发，支持 Windows 7 至 11 及主流 Linux 发行版。
*   **统一管控**：教师端可同时控制 30-100 个学生端，实现下发试题、监控进度、统计成绩。
*   **安全可靠**：通信内容加密，具备防作弊、防逆向、防重放攻击能力。
*   **界面现代**：采用 Avalonia UI 框架，提供新颖、流畅的用户体验。

## 2. 用户角色定义
| 角色 | 职责描述 |
| :--- | :--- |
| **教师端** | 作为服务端控制节点。负责题库管理、考试流程控制（开始/停止）、实时监控学生状态、接收成绩、数据导出及异常处理。 |
| **学生端** | 作为客户端执行节点。负责身份签到、接收指令、进行打字测试（练习/考试）、提交成绩及断线重连。 |

## 3. 功能需求详解
### 3.1 教师端功能
#### 3.1.1 系统与连接管理
*   **服务开启**：一键启动局域网监听服务，自动发现在线学生。
*   **座位/设备管理**：
    *   显示学生列表（IP、计算机名、学号、姓名、状态）。
    *   **设备绑定管理**：查看当前已绑定设备的学生列表（含绑定时间、剩余时长）。
    *   **强制解绑**：支持手动清除学生的“设备-账号”绑定关系，允许学生更换座位。
#### 3.1.2 题库与考试控制
*   **题库管理**：支持添加、编辑、删除测试文章（中文/英文/代码），支持 TXT 文件导入。
*   **考试模式选择**：
    *   **定篇测速**：下发固定文章，计算打字耗时。
    *   **限时冲刺**：设定时长（1-20分钟），计算打字数量。
    *   **纠错模式**：下发含错文本，考核修改能力（可选）。
*   **流程控制**：
    *   **开始考试**：锁定学生端界面，倒计时开始。
    *   **暂停/停止考试**：暂停界面冻结或强制结束测试。
    *   **试卷下发**：将试题加密传输至所有在线学生端。
#### 3.1.3 监控与统计
*   **实时监控**：以仪表盘形式展示所有学生状态（空闲/正在考试/已完成/离线）。
*   **异常预警**：当检测到疑似作弊行为（如速度异常波动、重放攻击尝试）时，在界面显著位置发出警报。
*   **成绩收卷**：自动接收学生端上报的成绩数据，进行解密验证。
*   **数据导出**：一键导出成绩单为 Excel (.xlsx) 格式，包含学号、姓名、速度、正确率、异常记录等字段。
### 3.2 学生端功能
#### 3.2.1 身份认证与绑定
*   **签到登录**：输入学号、姓名向教师端注册。
*   **防重登录（设备绑定）**：
    *   首次登录自动绑定当前设备指纹。
    *   若账号已绑定其他设备且未过期，拒绝登录并提示联系教师。
    *   绑定关系在 2 小时后自动失效。
#### 3.2.2 打字测试核心
*   **测试界面**：
    *   显示原文区与输入区，实时高亮对比（正确绿色，错误红色）。
    *   悬浮仪表盘显示剩余时间、实时速度。
*   **模式支持**：支持定篇测速、限时冲刺等多种模式。
*   **输入体验**：支持阻塞式输入或自由输入模式。
#### 3.2.3 成绩与通信
*   **实时上报**：考试过程中每隔 1-2 秒上报实时速度及状态。
*   **断线重连**：网络波动时自动尝试重连教师端，保证考试数据不丢失。
*   **成绩提交**：考试结束自动打包加密成绩数据发送至教师端。

## 4. 安全与防作弊设计
### 4.1 防作弊算法（学生端核心）
*   **屏蔽复制粘贴**：拦截 `Ctrl+V`、右键菜单及剪贴板操作，禁止鼠标拖拽文本。
*   **防止输入法预输入（物理时间差检测）**：
    *   监听每个字符的输入时间戳。
    *   若检测到一次事件输入多个字符，或连续字符时间差 < 30ms（超出人类生理极限），判定为“批量上屏”作弊。
    *   处理策略：对瞬间输入的字符仅计入前 1-2 个有效，其余丢弃，并标记异常上报教师端。
*   **速度异常监控**：计算滑动窗口平均速度，若瞬时速度远超正常范围，自动触发异常警报。
### 4.2 防重登录机制
*   **设备指纹**：采集 CPU ID + 网卡 MAC + 硬盘序列号生成唯一设备标识。
*   **绑定逻辑**：数据库存储“学号-设备指纹-绑定时间”映射表，2小时自动过期。
### 4.3 通信安全（防重放/防篡改）
*   **加密传输**：全链路采用 AES 加密传输数据。
*   **自定义二进制协议**：
    *   结构：`Header | Version | MsgType | Timestamp | Nonce | DataLen | Data | Signature | Tail`。
    *   **签名机制**：使用 HMAC-SHA256 对报文进行签名，防止数据篡改。
*   **防重放攻击**：
    *   报文携带毫秒级时间戳，接收端验证时间差（> 60秒视为过期）。
    *   报文携带随机数，接收端缓存近期 Nonce 值，拒绝重复 ID 的报文。
### 4.4 防逆向工程
*   **AOT 发布**：项目编译为 Native AOT 原生二进制文件，无 IL 中间语言，杜绝 ILSpy/dnSpy 等反编译工具直接查看源码。
*   **符号剥离**：发布版本不生成 PDB 调试符号，移除函数名与变量名信息。

## 5. 界面设计需求 (UI/UX)
### 5.1 设计风格
*   **框架**：Avalonia UI 11/12。
*   **主题**：现代扁平化设计，支持明暗主题切换。
*   **视觉效果**：高对比度易读性，适应机房投影展示；界面动画流畅，控件响应迅速。
### 5.2 教师端界面
*   **布局**：左侧导航栏（考生列表、题库、统计）+ 顶部工具栏 + 中间数据看板。
*   **看板视图**：网格化显示学生状态（灰色-离线，绿色闪烁-打字中，蓝色-已完成，红色-异常）。
*   **异常浮窗**：右侧独立区域滚动显示作弊嫌疑警报。
### 5.3 学生端界面
*   **沉浸模式**：考试时全屏置顶，覆盖 90% 屏幕，禁止窗口移动或最小化。
*   **打字区**：类似代码编辑器风格，行号显示，光标高亮，字体清晰（字号 18-20px）。
*   **仪表盘**：右上角悬浮圆形进度条，实时反馈时间与速度。

## 6. 技术选型与架构
### 6.1 核心架构
*   **运行时**：.NET 8 (LTS)。
*   **编译发布**：Native AOT (Ahead-of-Time)。
*   **架构模式**：MVVM (Model-View-ViewModel)。
### 6.2 关键组件选型
| 模块 | 技术选型 | 选型理由 |
| :--- | :--- | :--- |
| **UI 框架** | Avalonia UI 11 | 跨平台支持 Win/Linux，现代化 XAML 框架，支持 GPU 渲染。 |
| **通信协议** | TCP Async Socket + Protobuf | 异步 Socket 处理高并发 (100节点)；Protobuf 二进制序列化，体积小速度快。 |
| **序列化** | protobuf-net (Source Generator) | 使用源生成器模式，完美兼容 AOT 编译，避免反射。 |
| **数据导出** | MiniExcel | 轻量级 Excel 库，低内存占用，支持强类型写入，兼容 AOT。 |
| **数据存储** | SQLite + Dapper | 单文件数据库，跨平台，便于分发和迁移。 |
| **加密算法** | System.Security.Cryptography | .NET 内置库，支持 AES/RSA/HMAC-SHA256。 |
### 6.3 通信协议定义
采用自定义二进制协议，包含：
*   **Header/Tail**：魔术字标识包起止。
*   **Timestamp/Nonce**：用于防重放校验。
*   **Signature**：HMAC-SHA256 签名防篡改。
*   **Data**：Protobuf 序列化的业务数据。

## 7. 部署与交付
### 7.1 发布产物
*   **Windows 版本**：生成 `Teacher.exe` / `Student.exe` 及必要的运行时 DLL（VC++ 运行库需预装或打包）。
*   **Linux 版本**：生成原生 ELF 可执行文件。
### 7.2 兼容性要求
*   **Windows 7**：需安装特定补丁以支持 .NET 8，建议在镜像制作时完成环境配置。
*   **Windows 10/11**：原生支持。
*   **Linux**：支持 glibc 版本较新的发行版（如 Ubuntu 20.04+, CentOS 7+）。
