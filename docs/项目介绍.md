# TypeU 项目介绍

## 背景与目标

大学机房打字教学与考核长期依赖单机软件，缺少统一管理、实时监控和有效防作弊。TypeU 通过局域网互联，让教师端一键管控 30–100 个学生端的签到、下发试题、监控、收卷与导出。

**目标平台**：Windows 10 1607+/11、Linux x64
**UI**：Avalonia（跨平台、支持明暗主题）  
**发布**：Native AOT（降低反编译风险）

## 角色

| 角色 | 职责 |
| :--- | :--- |
| 教师端 | TCP 服务端：题库、考试控制、实时监控、收卷导出、设备解绑、局域网发现 |
| 学生端 | TCP 客户端：签到绑定、接收指令、打字测试、状态上报、断线重连 |

## 核心能力

### 教师端

- 一键启动监听；UDP 广播供学生端自动发现
- 学生列表与状态看板（空闲 / 考试中 / 已完成 / 离线 / 异常）
- 题库管理（中文 / 英文 / 代码，支持 TXT 导入）
- 考试模式：定篇测速、限时冲刺（1–20 分钟）、纠错模式
- 流程：开始 / 暂停 / 停止 / 重新考试；加密下发试题
- 成绩收卷校验与 Excel 导出（MiniExcel）
- 设备绑定查看与强制解绑；局域网一键扫描

### 学生端

- **默认单机模式**；配置教师 IP 后 Ping→连接；状态栏显示联网模式
- 仅开考后登录（学号+姓名）；同会话同设备可自动登录；登录后禁止自行退出
- 按重考次数多次测验并每次回传；考试结束后可自行登出清除本机信息
- 打字界面：原文/输入区实时高亮，悬浮仪表盘（时间 / 速度）

### 教师端

- 启动录入工号/姓名；未开考可巡视连接学生（模式/速度/是否最小化）
- 开考配置：时长、类型、题库、重考次数 1–5、完成后是否自由练习；会话持久化可恢复
- 结束考试后学生可自选登出；可单独「允许登出」；导出含各次成绩与最高成绩

### 安全与防作弊

1. **输入层**：屏蔽复制粘贴 / 拖拽；字符时间差与批量上屏检测；速度滑动窗口异常告警  
2. **设备层**：CPU ID + MAC + 硬盘序列号生成指纹，防多机重登  
3. **通信层**：AES-256-CBC + HMAC-SHA256；时间戳 + Nonce 防重放；教师端时间同步防改本地时钟  
4. **发布层**：Native AOT、符号剥离

## 架构概览

```
src/
├── TypeU.Core/           # 设备指纹、防作弊等核心逻辑
├── TypeU.Models/         # 实体与 DTO（protobuf-net）
├── TypeU.Network/        # 协议编解码、加签、TCP、UDP 发现
├── TypeU.Data/           # SQLite + Dapper
├── TypeU.Services/       # 教师 / 学生业务服务
├── TypeU.Teacher.GUI/    # 教师端 Avalonia
├── TypeU.Student.GUI/    # 学生端 Avalonia
└── TypeU.Tests/          # 单元 / 集成测试
```

依赖方向（单向）：

```
GUI → Services → Core / Network / Data / Models
Tests 可引用被测项目
```

命名空间统一前缀：`Larpx.PersonalTools.TypeU.`

## 技术选型

| 模块 | 选型 |
| :--- | :--- |
| 运行时 | .NET 10 |
| UI | Avalonia UI + CommunityToolkit.Mvvm |
| 通信 | TCP Async Socket + 自定义二进制包 |
| 序列化 | protobuf-net（反射模式，AOT 裁剪兼容） |
| 存储 | SQLite + Dapper |
| 导出 | MiniExcel |
| 日志 / DI | Serilog · Microsoft.Extensions.DependencyInjection |

### 报文结构（摘要）

```
Magic(0xAA55) | Version | MsgType | Timestamp | Nonce | DataLen | Data | Signature | Tail(0x55AA)
```

Data 为 Protobuf，经 AES 加密；Signature 为 HMAC-SHA256。
