# TypeU 二次开发指南

允许基于本仓库二次开发与分发，须遵守 [MIT License](../LICENSE)：在副本或实质性衍生作品中**保留** `Copyright (c) 2026 Larpx` 与许可全文。

## 环境

- [.NET 10 SDK](https://dotnet.microsoft.com/download)
- 推荐 IDE：Visual Studio 2022 / Rider / VS Code + C# 扩展
- 目标 OS：Windows 或 Linux x64（GUI 需图形环境）

```bash
dotnet --version   # 应 ≥ 10
dotnet build TypeU.slnx
dotnet test src/TypeU.Tests
```

## 仓库约定

| 项 | 约定 |
| :--- | :--- |
| 解决方案 | `TypeU.slnx`，项目在 `src/` |
| 命名空间 | 必须以 `Larpx.PersonalTools.TypeU.` 开头，并包裹全部代码 |
| 顶级语句 | 禁止；入口使用显式 `Main` |
| 公共属性 | `Directory.Build.props`（`net10.0`、Nullable、TreatWarningsAsErrors、ImplicitUsings=disable） |
| AOT | GUI 项目启用 `PublishAot` / `StripSymbols`；序列化避免反射，优先 Source Generator |

依赖只允许向下引用，勿让 Core/Data 反向依赖 GUI。

## 模块该改哪里

| 需求类型 | 优先位置 |
| :--- | :--- |
| 协议字段 / 报文类型 | `TypeU.Network`（`Protocol/`、`Messages/`） |
| 加解密、Nonce、时间校验 | `TypeU.Network/Security/` |
| TCP / UDP 发现 | `TypeU.Network/Tcp/`、`Discovery/` |
| 实体表结构、仓储 | `TypeU.Models`、`TypeU.Data` |
| 设备指纹、防作弊阈值 | `TypeU.Core` |
| 考试 / 题库 / 监控 / 成绩 | `TypeU.Services/Teacher/` |
| 登录 / 打字 / 上报 / 提交 | `TypeU.Services/Student/` |
| DI 注册 | `TypeU.Services/Bootstrapping/` |
| 界面与 ViewModel | 对应 `*.GUI`（MVVM） |

新增服务：实现逻辑 → 在 `ServiceCollectionExtensions` 注册 → GUI 通过构造注入使用 → 在 `TypeU.Tests` 补测试。

## 常用命令

```bash
# 调试运行
dotnet run --project src/TypeU.Teacher.GUI
dotnet run --project src/TypeU.Student.GUI

# 测试
dotnet test src/TypeU.Tests

# AOT 发布示例（按需调整 rid）
dotnet publish src/TypeU.Teacher.GUI -c Release -r win-x64 --self-contained
dotnet publish src/TypeU.Student.GUI -c Release -r linux-x64 --self-contained
```

发布产物：Windows 一般为可执行文件 + 依赖 DLL；Linux 为原生 ELF。机房镜像需注意 Win7 对较新 .NET 的兼容性（见需求文档部署章节）。

## 扩展建议（最小路径）

1. **新消息类型**：在 `MessageType` / Messages 增加定义 → Packet 编解码与业务处理 → 两端服务联动 → 测试往返与防重放。  
2. **新考试模式**：扩展 `ExamMode` 与教师控制流程 → 学生端 `TypingTestService` 计分逻辑 → UI 选项。  
3. **新页面**：Avalonia View + ViewModel → 导航接入现有主窗口 → 仅调用 Services，不直接写 SQL/Socket。

## 二次开发时请注意

- 通信包必须继续走加密与签名；勿为图省事绕过 `PacketCodec`。  
- 考试倒计时应以教师端时间同步为准，勿单独信任学生机系统时钟。  
- AOT 下避免运行时反射序列化；新增 Protobuf 类型需正确标注并走源生成。  
- 局域网 UDP 广播可能被交换机隔离，保留「手动输入教师端 IP」兜底。  
- 改动公共协议后，教师端与学生端须同步升级。
